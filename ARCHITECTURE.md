# Архитектура Med System

## Структура проекта

```
med-system/
├── main.py                  # Точка входа
├── build.bat                # Сборка exe (PyInstaller)
├── requirements.txt         # Зависимости
├── assets/icons/            # Иконки
└── app/
    ├── config.py            # ВСЕ настройки и пути — в одном месте
    ├── app.py               # Контроллер: навигация, связывание страниц с данными
    ├── ui.py                # Тема, общие виджеты (кнопки, тосты, маска дат)
    ├── validators.py        # Валидация форм и бизнес-правила по срокам
    ├── db/                  # СЛОЙ ДАННЫХ (только здесь есть SQL)
    │   ├── connection.py    # Подключение к БД (единая точка)
    │   ├── schema.py        # Создание таблиц, миграции, справочник МКБ
    │   ├── employees.py     # Сотрудники
    │   ├── students.py      # Студенты
    │   ├── groups.py        # Группы + перевод на следующий курс
    │   ├── medicines.py     # Лекарства + заказ партий
    │   ├── appeals.py       # Обращения
    │   └── icd.py           # Коды МКБ
    └── pages/               # Страницы интерфейса (по одной на экран)
```

## Правила слоёв

1. **pages/** не знают про SQL — получают данные через колбэки из `app.py`.
2. **db/** не знает про интерфейс — только чистые функции над данными.
3. Весь SQL живёт только в `app/db/`. Каждая функция — одна транзакция:
   `get_connection()` сам коммитит при успехе и откатывает при ошибке.
4. Настройки (пути, пороги, размеры окна) — только в `config.py`.

## Будущий переход на сетевую БД (PostgreSQL)

План, когда понадобится общая база для нескольких ПК:

1. Поднять PostgreSQL на одном компьютере/сервере в локальной сети.
2. Заменить `app/db/connection.py`: `sqlite3.connect` → `psycopg2.connect`
   (хост/порт/пароль добавить в `config.py`).
3. В запросах заменить плейсхолдеры `?` → `%s` (только внутри `app/db/`).
4. `schema.py`: `AUTOINCREMENT` → `GENERATED ALWAYS AS IDENTITY`,
   PRAGMA-миграции переписать под information_schema.

Остальное приложение (страницы, валидация, контроллер) менять не придётся —
интерфейс пакета `app.db` останется прежним. Синхронизация не нужна:
все ПК работают с одной базой и всегда видят актуальные данные.

**Важно:** не класть файл SQLite на сетевую папку — при одновременной
записи с двух ПК база может повредиться.
